# 失败案例修复

## 修复方法

先识别最主要的问题，再做单点修复，不要一次改所有东西。
默认仍按 `15 秒` 视频来重写。

## 常见问题映射

| 问题 | 常见原因 | 重写方向 |
| --- | --- | --- |
| 角色漂移 | 角色描述写太多，环境和特效抢主体 | 依赖参考图控角，压缩角色描述，转而强化动作和表情 |
| 动作发脏 | 同时写了多个互相打架的动作 | 压成一个连续动作，明确起点和结束点 |
| 镜头混乱 | 推拉摇移跟堆在一起 | 只留一种主镜头逻辑 |
| 像海报不像视频 | 静态形容词太多，过程信息太少 | 增加动作过程、镜头路径和结尾状态 |
| 环境不成立 | 只写了背景名词，没有接触关系 | 增加角色与桌边、货架、窗台、街道等接触点 |
| 环境抢戏 | 环境描述太多，动作卖点不清楚 | 环境压到 2 到 4 个锚点，把动作放回主体中心 |
| 没有情绪点 | 词很多，但没有单一情绪中心 | 锁定一个情绪，其他元素全部服务它 |
| 有剧情没卖点 | 写了前因后果，但没有一个记忆动作 | 删掉剧情推进，保留一个动作、一个反应、一个收尾 |
| 没有传播感 | 开头平、动作弱、结尾断 | 强化开头抓点、记忆动作和 loop 收尾 |

## 快速重写模板

当结果失败时，优先改成下面结构：

```md
单主体 + 单环境 + 单连续动作 + 单镜头逻辑 + 单情绪中心 + 明确的 15 秒节奏 + 明确结尾状态
```
