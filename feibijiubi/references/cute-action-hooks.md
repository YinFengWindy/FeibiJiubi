# 可爱动作卖点库

## 用法

当用户想要“更萌、更有动作感、更像短视频”，或者明确说不要走剧情时，优先先读这份文档。
先选一个主动作，再给它配一个表情、一个环境接触点和一个收尾，不要反过来先编故事。

## 动作卖点判断标准

优先选符合下面条件的动作：

- 一眼能看懂
- 2 到 5 秒内就能成立
- 近景或半身能拍清楚
- 能自然带出脸部反应
- 结尾能定格、回弹或 loop

## 动作家族

### 1. 看镜头类

代表动作：
- 歪头看镜头
- 身体微微前探
- 眨眼后甜笑

适合：
- 近景
- 第一眼抓人
- 低剧情需求

### 2. 托脸类

代表动作：
- 双手捧脸
- 单手托腮
- 托脸轻晃

适合：
- 教室、咖啡店、窗边、桌边
- 静态环境里出可爱感
- 表情戏

### 3. 抱道具类

代表动作：
- 抱杯子
- 抱枕蹭蹭
- 抱购物篮回头笑

适合：
- 需要接触关系
- 日常感和陪伴感
- 半身镜头

### 4. 出场类

代表动作：
- 小跳步入画
- 小跑后回头
- 从门边或货架后探出来

适合：
- 开场抓点
- 半身到中景
- 环境辨识度强的场景

### 5. 招呼类

代表动作：
- 挥手
- 比心
- 贴近镜头招呼

适合：
- 开头或结尾
- 传播感强
- 系列化内容

### 6. 害羞反应类

代表动作：
- 鼓脸
- 捂脸
- 轻轻别开视线

适合：
- 近景反应
- 单情绪短片
- loop 收尾

### 7. 道具展示类

代表动作：
- 拿手机看消息后抿嘴笑
- 拿电话贴脸卖萌
- 举着小牌子或饮料朝镜头示意

适合：
- 动作需要明确动机时
- 现实环境
- 半身机位

## 组合公式

默认按下面结构组合：

```md
一个主动作 + 一个表情变化 + 一个环境接触点 + 一个镜头逻辑 + 一个收尾机制
```

## 默认推荐顺序

用户没指定时，优先尝试：

1. 歪头看镜头
2. 双手捧脸或托腮
3. 抱道具蹭蹭
4. 小跳步回头
5. 挥手或比心

## 写法提醒

- 一个视频只保留一个主动作，不要把多个动作并列堆进去。
- 表情必须服务动作，不要让表情和动作分别在讲不同的事。
- 环境只负责托动作成立，不负责讲完整剧情。
- 如果动作不够清楚，先减环境和镜头，不要继续加设定。
