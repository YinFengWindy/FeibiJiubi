# 菲比啾比即梦视频提示词优化

## 目标

专注即梦的视频模式。
把“想法、草稿、失败案例”改写成更适合视频生成的提示词，而不是停留在图片式描述。

## 使用原则

- 默认输出中文提示词，必要的英文风格词和镜头词可保留。
- 优先保证主体一致性、动作清晰度和镜头可执行性。
- 优先做“少而准”的强化，不堆砌形容词。
- 面对萌系二创，优先放大情绪记忆点、动作记忆点和循环记忆点。
- 用户没有明确要求时，不擅自更换角色设定、服装、世界观或画幅风格。

## 触发信号

在以下场景使用本技能：
- 用户想把一句话点子改成即梦视频提示词。
- 用户已经有草稿，但生成结果“普通、散、像图片、没动感、没短视频味”。
- 用户想做菲比啾比风、Q版萌化、团子化、软弹治愈、轻循环角色短片。
- 用户描述的是“某角色很可爱地做一个动作或表情”的二创短视频。
- 用户需要把失败结果重写成更稳定的即梦视频提示词。

## 工作流

1. 先判断任务类型：
   - 点子直出
   - 草稿重写
   - 失败修复
   - 风格迁移

2. 再抽取核心槽位：
   - 主体
   - 场景
   - 动作
   - 镜头
   - 节奏
   - 情绪
   - 光线与质感
   - 风格锚点
   - 负面约束

3. 信息不全时，只追问最关键的 1 到 2 个点：
   - 主体是否固定
   - 想要什么情绪和风格
   - 更偏动作表演还是镜头氛围

4. 直接输出可用结果，不只给分析。

## 改写重点

### 1. 把静态描述改成视频描述

把“长什么样”改成“在做什么、怎么动、镜头怎么跟、节奏怎么推进”。

优先写清：
- 动作起点
- 动作持续方式
- 镜头移动方式
- 结尾停留或循环点

### 2. 保住主体一致性

对角色二创，始终优先：
- 单主体
- 单核心动作
- 单核心情绪
- 单主场景

除非用户明确要求，否则不要同时堆太多：
- 多角色互动
- 复杂场景切换
- 大量特效
- 多段剧情推进

### 3. 让“菲比啾比风”可执行

当用户提到“菲比啾比风”或相近表达时，优先理解为：
- 萌化
- Q版或团子化
- 软弹轻盈
- 低攻击性
- 治愈、轻松、可循环传播
- 强表情、弱剧情
- 动作简单但记忆点强

不要只写“可爱、治愈、萌”，要补成具体画面锚点：
- 圆润比例
- 轻微摇摆
- 软弹回弹
- 亮眼表情
- 柔和风感
- 奶油或果冻质感
- 结尾能自然回到开头

### 4. 让短视频更像短视频

优先强化：
- 开头 1 秒是否有抓点
- 动作是否一眼看懂
- 镜头是否服务动作
- 情绪是否集中
- 结尾是否适合 loop

如果用户只给一句话，默认采用：
- 单主体
- 单场景
- 单动作主线
- 轻镜头运动
- 强氛围收尾

## 输出格式

固定按下面结构输出：

```md
主提示词：
<一段可直接用于即梦视频的完整提示词>

分镜/镜头节奏版：
1. <开场镜头与动作>
2. <中段动作与镜头推进>
3. <结尾停留或循环收束>

负面提示词：
- <应避免的画面、动作、镜头或风格问题>

三档版本：
- 保守版：<更稳、更少元素、更强调一致性>
- 平衡版：<默认推荐，兼顾记忆点和稳定性>
- 强化版：<更强风格化、更强传播感，但风险更高>

优化说明：
- <本次改写强化了什么>
- <如果继续优化，下一步最值得调什么>
```

## 失败修复规则

当用户反馈结果不好时，优先按问题类型重写：

- 主体不稳定：
  - 收缩人物设定词。
  - 降低多余服装、环境、特效描述。
  - 提高“单主体 + 单动作 + 单镜头主线”的优先级。

- 动作混乱：
  - 把动作改成单段连续动作。
  - 去掉互相冲突的动词。
  - 明确“慢慢摇摆、轻轻转头、轻跳一下”这类节奏。

- 镜头太乱：
  - 只保留一种主镜头逻辑。
  - 避免同时出现推拉摇移跟的大杂烩。
  - 镜头词少而准。

- 画面像图片不像视频：
  - 增加动作过程、镜头路径、情绪变化和结尾状态。
  - 减少纯静态堆砌词。

- 没有短视频传播感：
  - 增加开头抓点。
  - 增加记忆动作或记忆表情。
  - 增加适合循环的结尾。

## 不要这样做

- 不要把图片提示词原样搬进视频模式。
- 不要同时堆太多风格词、特效词和镜头词。
- 不要在同一条提示词里塞入多个情绪中心。
- 不要在用户没要求时擅自改角色核心设定。
- 不要把“可爱、治愈、梦幻”这种空泛词当成最终结果，必须补成可执行的画面和动作。
